## Project Overview

**membran** is a React Native macOS desktop app ("layer I").

| | |
|---|---|
| Platform | macOS desktop (via `react-native-macos` by Microsoft) |
| Language | TypeScript + React Native 0.81 |
| Framework | **NOT Expo** ‚Äî bare React Native + `react-native-macos` |
| Min macOS | 12.0 (Monterey) |

> **Why not Expo?** Expo does not support macOS desktop apps. The correct
> tool for native macOS with React Native is Microsoft's
> [`react-native-macos`](https://github.com/microsoft/react-native-macos).

## First-Time Setup (macOS only)

You need a Mac with:
- macOS 12+ (Monterey or newer)
- Xcode 15+ with Command Line Tools (`xcode-select --install`)
- Node 22 LTS ‚Äî use nvm: `brew install nvm && nvm install 22`
- Ruby 3.3 ‚Äî use rbenv: `brew install rbenv ruby-build && rbenv install 3.3.6`
- CocoaPods ‚Äî **`brew install cocoapods`** (‚öÝÔ∏è never `sudo gem install cocoapods`
  on Apple Silicon ‚Äî it installs an x86 build that fails on arm64 Macs)
- Watchman: `brew install watchman`

```bash
git clone <repo-url>
cd membran
bash scripts/setup-macos.sh
```

That script runs `npm install`, `react-native-macos-init`, and `pod install`
in one shot. Only needs to be run once (or after adding a new native module).

## Running the App

You need **two terminals**.

**Terminal 1 ‚Äî Metro bundler (macOS)**
```bash
npm run start:macos
# Starts Metro on port 8082 (separate from iOS/Android bundler on 8081)
```

**Terminal 2 ‚Äî Build & launch**
```bash
npx react-native run-macos --scheme membran
```

Or open `macos/membran.xcworkspace` in Xcode and press **Run (‚åòR)**.

## Key Concepts for AI Assistants

### Platform files
React Native supports platform-specific files. For macOS overrides, use the
`.macos.tsx` suffix ‚Äî Metro will prefer it over the base file on macOS:

```
components/
‚îú‚îÄ‚îÄ Button.tsx          # shared / iOS / Android
‚îî‚îÄ‚îÄ Button.macos.tsx    # macOS-specific override
```

### Metro port 8082
The macOS bundler intentionally runs on port **8082** (`npm run start:macos`),
not 8081, so iOS and macOS development can run side-by-side.

### CocoaPods
All native dependencies are managed through CocoaPods in the `macos/` directory.
After adding a new npm package that has native code, run:
```bash
cd macos && pod install
```

### React Native APIs on macOS
Most React Native core APIs work on macOS. Notable differences:
- `StatusBar` is a no-op on macOS (no status bar on desktop)
- `SafeAreaView` works but insets behave differently
- Navigation libraries (React Navigation) generally support macOS
- Camera, location, and other sensor APIs need macOS entitlements in
  `macos/membran/membran.entitlements`

## Known Gotchas

### CocoaPods on Apple Silicon
Use `brew install cocoapods`, not `sudo gem install cocoapods`.
The gem version installs an x86_64 `ffi` native extension that fails at runtime
on M-series Macs with `dlopen: missing compatible arch`.

Verify you have the correct binary:
```bash
which pod          # should be /opt/homebrew/bin/pod
pod --version      # 1.14.x or 1.16.x ‚Äî avoid 1.15.0 and 1.15.1
```

### Version alignment
The minor version of `react-native` and `react-native-macos` **must match**.
Mixing (e.g., RN 0.81 + rn-macos 0.79) causes cryptic native build errors.
Both are pinned to `0.81.0` in this project's `package.json`.

### Library ecosystem
Not all React Native libraries support macOS. Before adding a dependency,
check [React Native Directory](https://reactnative.directory) with the
"macOS" filter enabled.

### New Architecture on macOS
The New Architecture (Fabric / TurboModules) is enabled by default starting
in React Native 0.76, but the macOS implementation lags behind iOS. If you
hit crashes or missing implementations, this is likely why. Check open issues
in [microsoft/react-native-macos](https://github.com/microsoft/react-native-macos).

## Git Workflow

### Branch naming

| Purpose | Pattern | Example |
|---------|---------|---------|
| AI-generated features | `claude/<description>-<session-id>` | `claude/add-auth-9mX0n` |
| Features | `feature/<description>` | `feature/settings-panel` |
| Bug fixes | `fix/<description>` | `fix/window-resize` |

- Default branch: **`main`**
- Never push directly to `main` ‚Äî use a PR
- AI branches **must** start with `claude/` and end with the session ID,
  or the push will fail with HTTP 403

### Push command
```bash
git push -u origin <branch-name>
```

On network failure: retry up to 4 times with exponential back-off
(2 s ‚Üí 4 s ‚Üí 8 s ‚Üí 16 s).

### Commit messages
Short imperative subject line, ‚â§72 characters:
```
Add settings sidebar component
Fix window not restoring last size
Remove unused import in App.tsx
```

## Coding Conventions

- **TypeScript** everywhere ‚Äî no plain `.js` files for app code
- **No default export hacks** ‚Äî named exports for utilities, default export
  only for the root component per React Native convention
- **Styles** via `StyleSheet.create()` ‚Äî never inline style objects
- **No dead code** ‚Äî delete unused files, imports, and variables
- **No premature abstraction** ‚Äî extract a helper only when a pattern
  repeats three or more times

### Security
- Validate all data at system boundaries (external APIs, user input)
- Secrets go in environment variables, never in source code
- Check `membran.entitlements` before requesting sensitive macOS permissions

## Adding Native Modules

1. `npm install <package>`
2. `cd macos && pod install`
3. If the package needs macOS entitlements, add them to
   `macos/membran/membran.entitlements`
4. Rebuild: `npx react-native run-macos --scheme membran`
